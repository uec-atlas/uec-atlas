# Contributing to UEC Atlas

UEC Atlasでは、Webサイトの改善やデータの追加など、様々な形での貢献を歓迎しています。以下は、プロジェクトへの貢献方法に関するガイドラインです。

なお、本プロジェクトに含まれるソースコードはMITライセンス、データはCC BY-NC-SA 4.0ライセンスで公開されており、これらに同意できる場合にのみ貢献を行ってください。

以下に記載のない事項については、[Webサイト](https://uec-atlas.org/)または[Wiki](https://github.com/uec-atlas/uec-atlas/wiki)を参照してください。


## 貢献の方法
### Issueの作成
バグやデータの改善点を見つけた場合は、Issueを作成して報告してください。

### Pull Requestの提出
コードやドキュメントの改善を行った場合は、Pull Requestを提出してください。変更内容を明確に説明し、関連するIssueがある場合はリンクしてください。

なお、データの追加や修正を行う場合は、Pull Requestの説明にその出典を明記してください。
また、特に必要な場合を除き、既存データに付与されたID(URI)を変更しないよう注意してください。


## プロジェクトの構成
UEC Atlasは、以下のような構成になっています。
- `data/`: 公開されているデータのベースとなるファイル
- `schema/`: データのスキーマ定義(LinkML)
- `scripts/`: データの取得処理を行うスクリプト
- `web/`: Webサイトのソースコード

なお、データ間の結合処理やRDFとしての出力処理など、データ加工の処理の一部は、Webサイトのビルドプロセスの一部として実行されます。`web/src/data/`および`web/src/pages/data/`ディレクトリ内のコードを参照してください。

## 開発環境の構築
開発コンテナー(Dev Container)を使用して、プロジェクトのルートディレクトリにある`.devcontainer/devcontainer.json`ファイルに基づいて開発環境を構築することができます。

手動で環境を構築する場合は、以下を参考にしてください。
- Node.js: v24以上
- pnpm: v10以上
- Python: v3.10以上

Webサイトの開発にはNode.jsとpnpmが、データの処理にはPythonが必要です。`package.json`および`requirements.txt`に記載された依存関係をインストールしてください。

`pnpm web dev`コマンドでWebサイトの開発サーバーを起動できます。
